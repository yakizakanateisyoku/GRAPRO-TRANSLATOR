# AGENTS.md

このリポジトリは Codex 自動レビューの対象です。
PR が作成されると、Codex がこのファイルの **Review guidelines** に従ってレビューを行います。

## Review guidelines

### P1（必ず指摘する：マージ前に修正必須）

**セキュリティ**
- 認証情報（API key、トークン、パスワード、Webhook URL、SSH 秘密鍵）がコード／コミット／ログ／環境変数のデフォルト値に直書きされていないか。
- 認可・権限チェックを通っていないルート／エンドポイント／コマンドがないか。
- 入力値検証なしで shell・SQL・eval に渡している箇所がないか。
- CORS・CSRF・SSRF 等の典型的な Web 脆弱性を新規に持ち込んでいないか。
- SSL/TLS 検証を意図せず無効化（`verify=False`, `check_hostname=False`, `-k` 等）していないか。意図的な場合はコメントで理由を明記しているか。

**個人情報・組織情報の漏れ**
- 実名（ユーザー本名）、勤務先の会社名・組織名、業務メールアドレス（@gmail.com 以外の本名を含むアドレス）がコード・コメント・ドキュメント・コミットメッセージ・テストデータに含まれていないか。
- 自宅住所、電話番号、クレジットカード番号、マイナンバー等が含まれていないか。
- ログ出力で個人を特定できる情報をマスクなしで吐いていないか。

### P2（推奨指摘：可能なら修正）

**コスト・リソース使用**
- 不要な外部 API 呼び出し（特に OpenAI / Anthropic / Cloudflare 課金 API）を増やしていないか。
- ポーリング間隔が不必要に短くないか（1 秒未満のループ、`while True` で sleep なし等）。
- N+1 クエリ、無限再帰、巨大な配列のメモリ展開を新規に持ち込んでいないか。
- キャッシュ可能なものをキャッシュせずに毎回計算／取得していないか。
- LXC・VM のリソース上限を踏むような重い処理（メモリ展開、ファイル一括読み込み）が入っていないか。

## Codex への補足指示

- レビューコメントは日本語で出してください。
- 指摘の優先度を `[P1]` / `[P2]` のプレフィックスで明示してください。
- 「問題なし」の場合も、何を確認したかを 1〜2 行で残してください。
